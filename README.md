# Object1
##医疗管理系统总体功能设计
《程序设计基础课程设计》（2025级）
一、需求分析
1. 需求包括：
数据规模：
至少容纳100名普通门诊患者、30名住院患者、20名医生、5个科室（每个科室至少3名医生）、3种病房（部分与科室关联）、20类药品（与科室关联）；需处理人名重名、名称长度等不规范输入的情况。
功能模块：
患者信息与医疗记录管理：维护患者个人信息，管理挂号、看诊、检查、住院。
住院病房与床位管理：管理病房类型，实现患者与床位的绑定/释放。
药房与药品管理：实现药品出入库、处方发药。
报表与查询管理：提供多视角查询（医护/管理/患者），支持按患者、医生、科室、药品名等维度统计查询。
二、总体设计
1. 系统架构
采用模块化分层设计：
数据持久化模块：负责所有实体数据的文件读写（加载/保存），保证数据持久化与完整性。
患者与医疗记录模块：管理患者信息与4类医疗记录，实现患者与医疗记录的关联查询。
病房与床位管理模块：管理病房、床位信息，实现床位分配/释放，统计床位使用状态。
药品与药房管理模块：管理药品信息，实现药品出入库、处方发药，校验库存合法性。
报表与查询模块：提供多视角数据查询与统计，生成业务报表；
2. 模块交互关系
患者模块与医疗记录模块：患者ID作为医疗记录，支持查询患者完整诊疗流程。
病房模块与床位模块：病房关联床位，床位状态变更同步更新病房空闲床位数。
药品模块与医疗记录模块：处方发药时，医疗记录关联药品信息。
三、详细设计
1. 数据结构设计
系统核心设计如下：

患者：ID(身份证号)、姓名、性别、年龄、联系方式、患者类型（门诊/住院）（此处要分开）、疾病、病房信息、床位信息
医生：ID(身份证号)、姓名、所属科室、联系方式、特长、职称
科室：科室名称、科室医生数共5个科室
病房：ID、病房类型、关联科室、总床位数、空闲床位数、门牌号、所属医生
床位：ID、所属病房、状态（空闲/占用）、关联患者ID
药品：ID、通用名、库存、单价、关联科室、生产商，成本价、售价
医疗记录：ID、关联患者ID、记录类型（挂号/看诊/检查/住院）、关联医生、时间记录、详细情况等
系统日志:（如何实现？）
2. 核心功能模块设计（函数模块）
（1）患者与医疗记录管理模块
患者信息管理：
增：录入患者基本信息，校验姓名长度、患者类型合法性
删/改：通过患者ID定位数据，修改后同步修改医疗记录。
查：支持按ID/姓名查询，重名时展示所有匹配患者信息。
医疗记录管理：
新增记录：选择患者、记录类型，录入日期、详情，关联对应医生。
查询记录：按患者ID查询，展示患者完整诊疗流程。
（2）病房与床位管理模块
病房管理：
增：录入病房类型、关联科室、总床位数，初始化空闲床位数=总床位数。
改：修改病房类型，更新关联床位的科室归属。
床位管理：
分配床位：选择住院患者与空闲床位，更新床位状态为占用，患者床位ID绑定，病房空闲床位数-1。
释放床位：患者出院后，更新床位状态为空闲，床位ID置为-1，病房空闲床位数+1。
（3）药品与药房管理模块
药品信息管理：
增：录入药品通用名、类别、单价、关联科室，校验名称长度。
查：支持按通用名查询，展示药品完整信息与库存。
出入库管理：
入库：增加药品库存，记录入库时间与数量。
出库：根据医疗记录扣减库存，校验库存是否充足。
处方发药：关联患者医疗记录，选择对应药品与数量，完成发药后更新库存并记录用药详情。
（4）报表与查询管理模块
多视角查询：
医护视角：自己的患者列表、所属的诊疗记录、某患者详细记录。
管理视角：统计所有患者信息、所有医生信息、药品库存、住院人数。
患者视角：查询个人医疗记录、个人药品记录、住院信息。
3. 数据的存储与加载
存储格式：采用txt文件存储
程序启动时：加载所有文件数据到内存链表，若文件不存在则初始化空数据。
程序退出/操作完成时：将内存中更新后的数据写回对应文件，保证数据一致性。
